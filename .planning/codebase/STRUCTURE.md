# Codebase Structure

**Analysis Date:** 2025-02-24

## Directory Layout

```
[project-root]/
├── backend/            # Express.js/TypeScript backend
│   ├── prisma/         # Database schema and migrations
│   ├── src/            # Application source code
│   │   ├── config/     # Server and service configuration
│   │   ├── controllers/# API request handlers
│   │   ├── middleware/ # Express middlewares (auth, audit, etc.)
│   │   ├── routes/     # API endpoint definitions
│   │   ├── services/   # Business logic and external integrations
│   │   ├── types/      # TypeScript type definitions
│   │   ├── utils/      # Shared helper functions
│   │   └── server.ts   # Backend entry point
│   ├── scripts/        # Maintenance and migration scripts
│   └── ssl/            # SSL certificates (if present)
├── frontend/           # Vite/React/TypeScript frontend
│   ├── src/            # Application source code
│   │   ├── components/ # Reusable UI components
│   │   ├── contexts/   # React contexts (Auth, Theme, etc.)
│   │   ├── hooks/      # Custom React hooks
│   │   ├── pages/      # Page-level components
│   │   ├── services/   # Frontend API client wrappers
│   │   ├── types/      # Frontend type definitions
│   │   └── main.tsx    # Frontend entry point
│   └── public/         # Static assets and service worker
├── plans/              # Design documents and implementation plans
└── scripts/            # Project-level management scripts
```

## Directory Purposes

**backend/src/controllers:**
- Purpose: Orchestrate HTTP requests by calling relevant services and returning responses.
- Contains: Classes or functions for managing firms, invoices, and user operations.
- Key files: `InvoiceController.ts`, `AuthController.ts`, `FirmController.ts`.

**backend/src/services:**
- Purpose: Contain the core business logic, decoupled from HTTP/Express concerns.
- Contains: Specialized service classes for ANAF integration, file storage, and background tasks.
- Key files: `AnafApiClient.ts`, `FileStorageService.ts`, `DownloadScheduler.ts`.

**frontend/src/pages:**
- Purpose: Higher-level components representing distinct views or routes in the application.
- Contains: Page components for Dashboard, Invoices, Settings, etc.
- Key files: `DashboardPage.tsx`, `InvoicesPage.tsx`, `FirmsPage.tsx`.

**frontend/src/services:**
- Purpose: Encapsulate API communication logic for use by frontend components.
- Contains: Axios-based or Fetch-based API client wrappers.
- Key files: `api.ts`, `authService.ts`, `invoiceService.ts`.

## Key File Locations

**Entry Points:**
- `backend/src/server.ts`: Initializes the Express server and starts all background services.
- `frontend/src/main.tsx`: Mounts the React application and provides context providers.

**Configuration:**
- `backend/prisma/schema.prisma`: Defines the database schema and models.
- `backend/src/config/`: Contains configuration for ANAF, Redis, and Database connections.

**Core Logic:**
- `backend/src/services/anaf/AnafApiClient.ts`: Handles all communication with the ANAF E-Factura API.
- `backend/src/services/scheduler/DownloadScheduler.ts`: Manages the automated invoice download process.

**Testing:**
- `backend/src/__tests__/`: Contains unit and integration tests for backend components.
- `backend/jest.config.js`: Configuration for the Jest test runner.

## Naming Conventions

**Files:**
- TypeScript Files: PascalCase for classes and components (`InvoiceController.ts`, `App.tsx`), camelCase for utility files and middleware (`auth.ts`, `logger.ts`).

**Directories:**
- camelCase for most directories (`controllers`, `middleware`, `services`).

## Where to Add New Code

**New Feature (e.g., Reports):**
- Primary code: `backend/src/controllers/ReportController.ts` and `backend/src/services/ReportService.ts`.
- Routes: `backend/src/routes/reports.ts`.
- Tests: `backend/src/__tests__/reports.test.ts`.

**New Frontend Page:**
- Page Component: `frontend/src/pages/ReportsPage.tsx`.
- Route definition: `frontend/src/App.tsx`.
- API calls: `frontend/src/services/reportService.ts`.

**Utilities:**
- Shared helpers: `backend/src/utils/` for backend or `frontend/src/utils/` (if it exists, otherwise `frontend/src/services/`) for frontend.

## Special Directories

**backend/prisma/migrations/:**
- Purpose: Contains database schema evolution scripts generated by Prisma Migrate.
- Generated: Yes (automatically by `prisma migrate dev`).
- Committed: Yes.

**backend/node_modules/ & frontend/node_modules/:**
- Purpose: External dependencies installed via npm/package manager.
- Generated: Yes.
- Committed: No.
