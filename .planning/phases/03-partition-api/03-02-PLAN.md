# Plan: Partition State & Audit Log

Implement detailed partition state reporting and paginated audit logs. Ensure existing arm/disarm routes follow the established API response standards.

## Tasks

<task id="03-02-01" requirement="API-06, PART-03">
Define `PartitionStateRead` schema in `app/partitions/schemas.py` that includes:
- Partition overall state
- List of member cameras with their detection status (found in snapshot or current ISAPI if available)
- Camera disarmed refcounts
</task>

<task id="03-02-02" requirement="API-06, PART-03">
Implement `GET /api/partitions/{id}/state` route in `app/partitions/routes.py` and its corresponding service logic in `service.py`. This endpoint provides a deep-dive into why a partition is in its current state (e.g., partial disarm).
</task>

<task id="03-02-03" requirement="API-07, DATA-09">
Implement paginated audit log endpoint `GET /api/partitions/{id}/audit` in `app/partitions/routes.py`. Ensure it supports `limit`, `offset` and returns a metadata-wrapped response (total, page, etc.).
</task>

<task id="03-02-04" requirement="API-01, API-05">
Review and refactor `POST /api/partitions/{id}/disarm` and `POST /api/partitions/{id}/arm` routes to ensure they use the `APIResponse` envelope consistently and handle errors gracefully according to the new schema standards.
</task>

<task id="03-02-05" requirement="API-01, API-06, API-07">
Update `tests/test_partitions.py` with tests for the state and audit log endpoints. Verify pagination on the audit log.
</task>

## Verification Criteria

### Automated Tests
- `pytest tests/test_partitions.py`
- Verify `GET /api/partitions/{id}/state` returns per-camera detection status and refcounts.
- Verify `GET /api/partitions/{id}/audit` returns paginated results with metadata.
- Verify Arm/Disarm endpoints return `APIResponse` envelopes.

### must_haves
- [ ] Audit log is separate from partition detail.
- [ ] State endpoint includes per-camera disarm refcounts.
- [ ] Audit log supports pagination (total, limit, offset).
