# Plan: Partition CRUD & Soft Delete

Implement full CRUD for partitions including soft-delete and deletion guard logic. Establish the core schema and service layer for managing partition metadata and camera membership.

## Tasks

<task id="03-01-01" requirement="PART-01, DATA-04">
Add `deleted_at: Mapped[datetime | None]` to `Partition` model in `app/partitions/models.py`. Update any existing queries in `service.py` (if any, although we're adding them now) to filter out soft-deleted partitions.
</task>

<task id="03-01-02" requirement="PART-01, PART-03, PART-04, API-04, API-09">
Define Partition schemas in `app/partitions/schemas.py`:
- `PartitionCreate`: name, description, location_id, auto_rearm_minutes, alert_if_disarmed_minutes, camera_ids
- `PartitionUpdate`: name, description, auto_rearm_minutes, alert_if_disarmed_minutes
- `PartitionRead`: id, name, description, location_id, auto_rearm_minutes, alert_if_disarmed_minutes, state, created_at
- `PartitionCameraSync`: camera_ids (for replacing membership)
</task>

<task id="03-01-03" requirement="PART-01, PART-02, PART-03, PART-04, PART-05, API-04">
Implement CRUD operations in `app/partitions/service.py`:
- `create_partition`: Creates Partition and initial PartitionState.
- `get_partitions`: Returns all non-deleted partitions with their current state.
- `get_partition_detail`: Returns partition with member cameras (NVR info included).
- `update_partition`: Updates metadata.
- `sync_partition_cameras`: Replaces camera membership. Validates location-bound (cameras must be in partition's location).
- `delete_partition`: Soft-delete. Blocks if state is `disarmed` or `partial`.
</task>

<task id="03-01-04" requirement="API-04, API-09">
Expose CRUD routes in `app/partitions/routes.py`:
- `POST /api/partitions`
- `GET /api/partitions`
- `GET /api/partitions/{id}`
- `PATCH /api/partitions/{id}`
- `DELETE /api/partitions/{id}`
- `PUT /api/partitions/{id}/cameras` (sync/replace)
</task>

<task id="03-01-05" requirement="API-09">
Create `tests/test_partitions.py` and implement unit tests for all CRUD operations, including soft-delete and the deletion guard.
</task>

## Verification Criteria

- Verify `PUT /api/partitions/{id}/cameras` replaces cameras and validates location.
