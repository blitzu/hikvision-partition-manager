<objective>
Implement Location CRUD and NVR CRUD endpoints with Fernet password encryption. This plan delivers the first working API surface that operators can actually use. The password security requirement (NVR-06) is enforced structurally — the NVRRead Pydantic schema has no password field, making leakage impossible at the serialization layer.

Purpose: Operators need to register their NVR infrastructure before any camera operations can happen. This plan establishes the two foundational domains and their security contract.

Output: POST/GET /api/locations, POST/GET /api/locations/{id}/nvrs — all returning the standard response envelope. Tests confirm encryption and password exclusion.
</objective>

    ```python
    import uuid
    from datetime import datetime
    from zoneinfo import ZoneInfo, ZoneInfoNotFoundError
    from pydantic import BaseModel, ConfigDict, field_validator

    class LocationCreate(BaseModel):
        name: str
        timezone: str

        @field_validator("timezone")
        @classmethod
        def validate_timezone(cls, v: str) -> str:
            try:
                ZoneInfo(v)
            except ZoneInfoNotFoundError:
                raise ValueError(f"Unknown timezone: {v!r}")
            return v

    class LocationRead(BaseModel):

        name: str
        timezone: str
        created_at: datetime

        model_config = ConfigDict(from_attributes=True)
    ```

    Create app/nvrs/schemas.py (Pattern 8 from RESEARCH.md — field exclusion via omission, not exclude=):
    ```python
    class NVRCreate(BaseModel):
        name: str
        ip_address: str
        port: int = 8000
        username: str
        password: str   # plaintext on create only — NEVER returned

    class NVRRead(BaseModel):
        id: uuid.UUID
        name: str
        ip_address: str
        port: int
        username: str
        location_id: uuid.UUID
        status: str
        last_seen_at: datetime | None
        created_at: datetime
        # NO password field — not password, not password_encrypted
        # Exclusion is structural: the field does not exist on this schema

        model_config = ConfigDict(from_attributes=True)
    ```

    Create app/nvrs/routes.py:
    - APIRouter(prefix="/api", tags=["nvrs"])
    - POST "/locations/{location_id}/nvrs":
      1. Verify location exists (db.get(Location, location_id)); if not, return APIResponse(success=False, error="Location not found")
      2. encrypted = encrypt_password(body.password)  — encrypt BEFORE any DB call
      3. Create NVRDevice ORM object with password_encrypted=encrypted (body.password is NOT stored anywhere else)
      4. db.add(nvr), await db.commit()
      5. Return APIResponse[NVRRead](success=True, data=NVRRead.model_validate(nvr))
    - GET "/locations/{location_id}/nvrs":
      1. Verify location exists; if not, return APIResponse(success=False, error="Location not found")
      2. Select all NVRDevice where location_id=location_id
      3. Return APIResponse[list[NVRRead]](success=True, data=[NVRRead.model_validate(n) for n in nvrs])

    Security constraint (NVR-06): The string `body.password` must NEVER appear in:
    - Any log statement
    - Any exception message format string
    - Any dict/JSON serialization
    Catch httpx errors using `type(exc).__name__` only (not str(exc)) to prevent credential leakage via exception messages.
