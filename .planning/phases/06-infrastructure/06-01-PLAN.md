<task type="auto">
  <name>Task 1: Write multi-stage Dockerfile with non-root user</name>
  <files>Dockerfile</files>
  <action>
Create a two-stage Dockerfile:

**Stage 1 (builder):** `FROM python:3.12-slim AS builder`
- WORKDIR /build
- COPY pyproject.toml .
- COPY app/ app/
- RUN pip install --no-cache-dir --prefix=/install .

**Stage 2 (runtime):** `FROM python:3.12-slim`
- Create non-root user: `RUN useradd --create-home --uid 1000 appuser`
- WORKDIR /app
- Copy installed packages from builder: `COPY --from=builder /install /usr/local`
- Copy source with correct ownership: `COPY --chown=appuser:appuser . .`
- Switch user: `USER appuser`
- Expose port 8000
- CMD: `["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]`

Critical details:
- Use `python:3.12-slim` (NOT alpine — asyncpg requires glibc)
- `--host 0.0.0.0` is required; 127.0.0.1 (uvicorn default) is unreachable from outside the container
- `COPY --chown=appuser:appuser . .` copies the entire project root including alembic/ and alembic.ini — required because lifespan runs migrations at startup via `asyncio.to_thread(_run_migrations)` which calls `Config("alembic.ini")`
- `--no-cache-dir` in pip reduces image size
- Do NOT add a HEALTHCHECK instruction in the Dockerfile — health check is defined in docker-compose.yml

<task type="auto">
  <name>Task 2: Write docker-compose.yml and update .env.example</name>
  <files>docker-compose.yml, .env.example</files>
  <action>
**docker-compose.yml:**

```yaml
services:
  db:
    image: postgres:16
    environment:
      POSTGRES_USER: appuser
      POSTGRES_PASSWORD: apppassword
      POSTGRES_DB: partitions
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U appuser -d partitions"]
      interval: 5s
      timeout: 5s
      retries: 10

  app:
    build: .
    env_file: .env
    environment:
      # Override DATABASE_URL so the app container reaches the db service (not localhost)
      DATABASE_URL: postgresql+asyncpg://appuser:apppassword@db:5432/partitions
    ports:
      - "8000:8000"
    depends_on:
      db:
        condition: service_healthy
    stop_grace_period: 35s

volumes:
  pgdata:
```

Key decisions per research and CONTEXT.md:
- `condition: service_healthy` (not just `depends_on: db`) prevents migrations from running before postgres accepts connections — this is the zero-manual-step guarantee
- `stop_grace_period: 35s` = 30s app graceful shutdown timeout + 5s buffer (per CONTEXT.md locked decision)
- DATABASE_URL override in `environment:` block takes precedence over the `.env` file value, so `docker compose up` works without manually editing .env — hostname `db` is the service name DNS
- Use `pgdata` as the volume name

**Update .env.example** to document all required vars (INFRA-03). The current .env.example is missing ALERT_WEBHOOK_URL and POLL_INTERVAL_SECONDS. Rewrite it as:

