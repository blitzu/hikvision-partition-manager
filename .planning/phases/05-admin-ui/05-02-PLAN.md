Add a helper function in `app/ui/routes.py` that fetches all NVRs across all locations along with their currently-synced cameras from the DB (no ISAPI call). This is used on page load for the camera selector:
```python
async def _get_nvrs_with_cameras(db: AsyncSession) -> list[dict]:
    """Return list of {nvr, cameras} dicts for the camera selector."""
    from sqlalchemy import select
    from app.nvrs.models import NVRDevice
    from app.cameras.models import Camera
    result = await db.execute(select(NVRDevice))
    nvrs = result.scalars().all()
    out = []
    for nvr in nvrs:
        cam_result = await db.execute(select(Camera).where(Camera.nvr_id == nvr.id))
        cameras = cam_result.scalars().all()
        out.append({"nvr": nvr, "cameras": cameras})
    return out
```

`GET /ui/nvrs/{nvr_id}/cameras/sync` (calls ISAPI sync then returns partial):
- Calls `GET /api/nvrs/{id}/cameras/sync` internally — actually, call the service directly. Import the sync logic. Simplest: make an internal HTTP call to `http://localhost:8000/api/nvrs/{nvr_id}/cameras/sync` via httpx. But simpler still: duplicate the camera query after sync by calling the camera sync service directly.
- Actually, call the existing route handler logic: import `sync_cameras` function from `app.cameras.routes` is not ideal since it's a handler. Instead, inline the camera DB query after calling `isapi_client.get_camera_channels()` is too complex. Use the simplest approach: make an internal httpx GET to the API endpoint. But that adds a dependency on the server being up.
- Best approach: add a thin helper in `app/cameras/service.py` (create it if absent) with `async def get_cameras_for_nvr(nvr_id, db) -> list[Camera]` that just queries DB. Then for the sync button, redirect to `GET /api/nvrs/{nvr_id}/cameras/sync` via HTMX and return the partial by calling the sync API internally via httpx.
- Simpler approach: The Sync button in `nvr_camera_section.html` uses `hx-get="/api/nvrs/{{ nvr.id }}/cameras/sync"` which returns JSON, not HTML. So instead, use `hx-get="/ui/nvrs/{{ nvr.id }}/cameras/sync"` which is a UI route that calls the API sync endpoint via httpx internally and then returns the HTML partial.

Implement `GET /ui/nvrs/{nvr_id}/cameras/sync`:
```python
@ui_router.get("/ui/nvrs/{nvr_id}/cameras/sync")
async def nvr_cameras_sync_partial(
    nvr_id: uuid.UUID,
    selected: str = Query(default=""),
    request: Request = None,
    db: AsyncSession = Depends(get_db),
):
    import httpx
    async with httpx.AsyncClient() as client:
        resp = await client.get(f"http://localhost:8000/api/nvrs/{nvr_id}/cameras/sync")
    # Regardless of sync result, query cameras from DB for the partial
    from sqlalchemy import select
    from app.cameras.models import Camera
    from app.nvrs.models import NVRDevice
    nvr = await db.get(NVRDevice, nvr_id)
    cam_result = await db.execute(select(Camera).where(Camera.nvr_id == nvr_id))
    cameras = cam_result.scalars().all()
    selected_ids = {uuid.UUID(s) for s in selected.split(",") if s}
    return templates.TemplateResponse("partials/nvr_camera_section.html", {
        "request": request, "nvr": nvr, "cameras": cameras, "selected_camera_ids": selected_ids,
    })
```
