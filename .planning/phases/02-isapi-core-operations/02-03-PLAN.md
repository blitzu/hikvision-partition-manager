<objective>
Implement the arm operation: refcount decrement for all member cameras, conditional ISAPI restoration and snapshot deletion for cameras reaching refcount 0, partition state update to "armed", and audit log entry.

Purpose: Arming is what makes the multi-partition refcount logic meaningful. A camera shared by two disarmed partitions must not re-enable detection until both partitions are armed. This plan implements that conditional restore with snapshot cleanup.
Output: arm_partition() service function, POST /api/partitions/{id}/arm endpoint wired into the existing partitions router, integration tests covering single-partition restore, multi-partition stay-disarmed, already-armed idempotency, and error cases.
</objective>

    **app/partitions/service.py** — Append `arm_partition` async function (do NOT touch `disarm_partition`):

    Algorithm:
    1. Load partition; if not found raise HTTPException 404.
    2. Load PartitionState. Idempotency check: if state == "armed", return ArmResponse(cameras_restored=0, cameras_kept_disarmed=0).
    3. Load all cameras in partition via PartitionCamera join → Camera.
    4. For each camera (can use asyncio.gather grouped by nvr_id for parallel execution, same pattern as disarm):
       a. Load CameraDisarmRefcount for camera_id. If not found or partition_id not in disarmed_by_partitions, skip this camera (it was never disarmed by this partition).
       b. Remove this partition_id from disarmed_by_partitions. Update the record.
       c. If len(disarmed_by_partitions) == 0 after removal (refcount reaches 0):
          - Load CameraDetectionSnapshot for (camera_id, partition_id).
          - For each detection_type in snapshot_data: call ISAPIClient.put_detection_config(camera.channel_no, detection_type, xml_body) where xml_body is snapshot_data[detection_type] — restore the original XML verbatim.
          - On httpx error: record PartitionError, increment error count, skip deletion.
          - On success: delete CameraDetectionSnapshot record.
          - Increment cameras_restored.
       d. If len(disarmed_by_partitions) > 0 after removal (refcount stays > 0):
          - Do NOT call ISAPI. Camera stays disabled.
          - Increment cameras_kept_disarmed.
    5. Set PartitionState.state = "armed", last_changed_by = armed_by, last_changed_at = now(UTC).
    6. ARM-05: Set scheduled_rearm_at = None (cancel any pending scheduled rearm — actual job cancellation deferred to Phase 4, just clear the field here).
    7. Append PartitionAuditLog(action="arm", performed_by=armed_by, audit_metadata={"cameras_restored": N, "cameras_kept_disarmed": M}).
    8. db.commit(); return ArmResponse(cameras_restored=N, cameras_kept_disarmed=M, errors=errors_list).
