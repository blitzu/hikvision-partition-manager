    - put_detection_config(channel_no, detection_type, xml_body): PUT the detection endpoint with Content-Type text/xml; same retry logic; return None on 200; raise on non-200 or second timeout
    - detection_type values are one of: "MotionDetection", "LineDetection", "FieldDetection", "shelteralarm"
    - URL pattern: {base_url}/ISAPI/Smart/{detection_type}/channels/{channel_no}
    - Retry: on httpx.TimeoutException ONLY — retry once. Non-timeout errors (4xx, 5xx) raise immediately, not retried.
    - Test: GET success returns XML string
    - Test: PUT success returns None
    - Test: GET timeout retries once, second timeout raises
    - Test: GET non-timeout error (e.g. 404) raises without retry (assert call count == 1)
    - Test: PUT timeout retries once then raises
  </behavior>

    Extend app/isapi/client.py — add these methods to ISAPIClient without restructuring the class or changing existing methods:

    1. Add a private `_detection_url(channel_no, detection_type)` helper that returns the URL string.

    2. Add `async def get_detection_config(self, channel_no: int, detection_type: str) -> str`:

       - Use `httpx.AsyncClient(**self._client_kwargs)` in async context manager

       - Same retry logic as get_detection_config
