<objective>
Implement the disarm operation: NVR pre-check, per-camera detection snapshot (skip if existing), parallel ISAPI disable writes, refcount increment, partition state update, and audit log entry.

Purpose: This is the primary value operation — disarming a partition is what operators trigger when a physical area needs to be unsecured for access. Without this, the system cannot fulfill its core function.
Output: POST /api/partitions/{id}/disarm endpoint backed by disarm_partition() service function; integration tests covering the success path, NVR pre-check failure, idempotent re-disarm, and multi-partition snapshot protection.
</objective>

  <action>
    **app/partitions/service.py** — Create with `disarm_partition` async function:

    Algorithm:
    1. Load partition; if not found raise HTTPException 404.
    2. Load or create PartitionState for the partition.
    3. Idempotency check: if state already "disarmed", return DisarmResponse(cameras_disarmed=0, cameras_kept_disarmed_by_other_partition=0).
    4. Load all cameras in partition via PartitionCamera join → Camera. If no cameras, transition state to "disarmed" and return.
    5. Load all NVRs for those cameras. NVR pre-check: call ISAPIClient(nvr).get_device_info() for each distinct NVR. On ANY failure (exception or HTTP error): set partition state to "error" with error_detail, append audit log, return APIResponse(success=False, error=...). Update nvr.last_seen_at and nvr.status="online" on each success.
    6. Per camera, in parallel grouped by nvr_id using asyncio.gather:
       a. Check if CameraDetectionSnapshot exists for (camera_id, partition_id). If yes, skip snapshot capture (DARM-04).
       b. If no snapshot: GET all 4 detection types. Save only the ones that return HTTP 200 (skip 404/other errors — camera may not support that type). Store as snapshot_data = {detection_type: xml_string, ...}.
       c. Insert CameraDetectionSnapshot record (skip if already exists — idempotent).
       d. For each detection type in snapshot where enabled=true in the XML: PUT the modified XML with enabled set to false. Parse XML with xml.etree.ElementTree; find the `enabled` element and set text to "false"; serialize back to string. Only PUT types that were originally enabled=true.
       e. Update or create CameraDisarmRefcount: append partition_id to disarmed_by_partitions array if not already present.
       f. On any httpx error for a camera: record in errors list; camera contributes to cameras_disarmed=0 for that camera.
    7. Determine final state: if all cameras succeeded → "disarmed"; if some failed → "partial"; update PartitionState.
    8. Set PartitionState.last_changed_by = disarmed_by, last_changed_at = now(UTC).
    9. If partition.auto_rearm_minutes is set: set scheduled_rearm_at = now(UTC) + timedelta(minutes=auto_rearm_minutes). (Job scheduling deferred to Phase 4 — just set the field here, DARM-07 is time-field only for now.)
    10. Append PartitionAuditLog(action="disarm", performed_by=disarmed_by, audit_metadata={"reason": reason, "cameras_disarmed": N}).
    11. db.commit(); return DisarmResponse.

    XML enabled-check helper: parse xml_text with ET.fromstring, find element with tag ending in "enabled" (strip namespace), check .text == "true". To disable: set .text = "false", serialize with ET.tostring(root, encoding="unicode").
