## Cross-Phase Integration Findings

### Finding 1: `settings.BASE_URL` Ignored — UI Proxy Calls Fragile

**Files:** `app/ui/routes.py:444`, `:498`, `:523`
**Affected requirements:** UI-04, NVR-01 (UI path), NVR-03 (UI path)
**Severity:** Tech debt (works in local dev; breaks in Docker Compose port change or reverse proxy)

Three UI httpx self-calls construct URLs as `http://localhost:8000/...` (string literals) instead of reading `settings.BASE_URL`. In Docker Compose, `localhost` resolves to the container itself (works only because uvicorn listens on `0.0.0.0:8000`). Any future port change, TLS termination at proxy, or alternate hostname silently breaks: NVR connectivity test (UI), per-NVR camera sync on partition form, and NVR create form submission. `settings.BASE_URL` exists in `app/core/config.py:8` as `"http://localhost:8000"` for exactly this purpose but `ui/routes.py` never imports `settings`.

### Finding 2: `POLL_INTERVAL_SECONDS` Inert Config Setting

**Files:** `app/core/config.py:12`, `app/main.py:92`, `app/main.py:99`, `.env.example:15`
**Affected requirements:** JOB-02, JOB-03, INFRA-03
**Severity:** Tech debt (misleading documentation; operators cannot configure poll intervals)

`POLL_INTERVAL_SECONDS: int = 300` is declared in Settings, documented in `.env.example`, and validated by `test_env_example_has_all_vars` — but neither scheduler registration in `app/main.py` reads it. `stuck_disarmed_monitor` is registered with `IntervalTrigger(minutes=5)` (hardcoded) and `nvr_health_check` with `IntervalTrigger(seconds=60)` (hardcoded). Operators who set `POLL_INTERVAL_SECONDS=600` in `.env` see no behavioral change.
